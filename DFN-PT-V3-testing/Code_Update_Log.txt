ReactiveSoluteTransport Code Log
Last updated: 2026-03-26

==================================================
1. Current Purpose of the Code
==================================================
This project is a C++ program for simulating reactive solute transport in fractured porous media.

Based on the current contents of `Code/src` and `Input`, the main workflow is:
1. Read domain, simulation, and DFN input file names;
2. Build the discrete fracture network (DFN) and computational domain;
3. Compute flow in the fracture network and extract the connected backbone;
4. Perform particle transport simulation with matrix diffusion and injection control;
5. Update fracture aperture during transport according to the reaction time step;
6. Output arrival-time statistics, particle position snapshots, and initial/updated DFN files.

Important characteristics of the current version:
- The main entry point is `Code/src/PERFORM.cpp`;
- Input files are selected through `Input/File_names.txt`;
- Results are written to the `Output/` directory;
- The code already includes chemistry-coupled aperture updates based on particle throughput and reaction time step;
- The code already supports staged outputs such as `DFN_step******.txt` and `particle_positions_t******.csv`.

==================================================
2. Meaning of Main Directories and Files
==================================================

[Root level]
Code/
Main source code directory. The current active development version should be considered `Code/src`.

Input/
Input parameter directory. It controls the domain, simulation, and DFN inputs.

Output/
Output directory. It stores DFN results, CDF/PDF files, particle position snapshots, and other simulation outputs.

test_case_matrix.xlsx
Spreadsheet used for test cases or case organization.

[Input related]
Input/File_names.txt
Entry file that specifies the 3 input files used by the current run, in this order:
- Domain file
- Simulation file
- DFN file

Input/Domain_files/
Contains domain size, matrix diffusion coefficient, porosity, and left/right hydraulic head parameters.

Input/Simulation_files/
Contains simulation control parameters such as:
- number of particles `nb_part`
- transfer probability parameter `proba_transfer`
- `simu_option` (infinite/finite matrix)
- `t_min`, `t_max`, `Nt`
- `seed_simu`
- `t_injection`
- `output_interval`
- `reaction_dt`

Input/DFN_files/
Contains fracture network input files. The current code can read DFN data from file and still keeps interfaces for other generation options.

[Main packages in Code/src]
Code/src/PERFORM.cpp
Program entry point. It connects parameter reading, DFN construction, flow computation, backbone extraction, particle transport, and result output.

Code/src/Input_Output/
Input/output module.
Main responsibilities:
- read parameter files;
- define boundary conditions;
- generate result files such as `cdf.txt` and `pdf.txt`.
Key files:
- `Parameters.h` / `Parameters.cpp`: read main input parameters;
- `BoundaryConditions.h` / `.cpp`: define boundary conditions;
- `Results.h` / `.cpp`: post-process and write result files.

Code/src/Domain_Definition/
Geometry and fracture network definition module.
Main responsibilities:
- define the simulation domain;
- define `FractureMesh` and `NetworkMeshes`;
- compute nodes, intersections, and fracture connectivity;
- compute and update hydraulic properties;
- support backbone extraction and DFN file output.
Key files:
- `Domain.h` / `.cpp`: domain boundary and geometric checks;
- `FractureMesh.h` / `.cpp`: discretized fracture element;
- `NetworkMeshes.h` / `.cpp`: complete fracture network;
- `HydraulicProperties.h` / `.cpp`: length, aperture, transmissivity;
- `DFNComputation.h` / `.cpp`: DFN-related computation support.

Code/src/Transport/
Core particle transport module.
Main responsibilities:
- particle injection;
- transport inside fractures and transfer between fractures;
- matrix diffusion;
- record `arrival_times`;
- write particle position snapshots;
- update aperture and network state at each `reaction_dt`.
Key files:
- `Transport.h` / `.cpp`: main transport workflow;
- `Particle.h` / `.cpp`: particle state definition;
- `Projection.cpp`: projection between fractures;
- `Transfer.cpp`: transfer probability and transfer time calculations.

Code/src/Chemistry/
Chemistry and aperture update module.
Main responsibilities:
- compute aperture changes from reaction time step and particle throughput;
- provide `ComputeAperture`, `ComputeDeltaAperture`, and related functions;
- support the reactive transport coupling.

Code/src/Flow/
Flow computation module.
The public header is currently minimal, but the general velocity and hydraulic computations are coupled with the logic in `NetworkMeshes` and flow-related routines.

Code/src/Visualisation/
Visualization and display support module.
Main responsibilities:
- display the fracture network;
- display segment-based results;
- help inspect DFN and transport behavior.

Code/src/Utilitaries/
Low-level utility module.
Main responsibilities:
- points, segments, and geometric structures;
- linear systems;
- random number utilities;
- constants;
- general numerical helper functions.
This is the shared base library used by several core modules.

Code/src/RngStream/
Random stream implementation used for stochastic sampling in particle transport.

[Other directories]
Code/Debug/
Build output directory containing object files, makefiles, and executable artifacts. It should not be treated as the main source of development documentation.

Code/Backup0318/ and Code/Backup0318_unmodified/
Historical backup directories. They are useful for comparison or rollback reference, but they are not the current main development target.
